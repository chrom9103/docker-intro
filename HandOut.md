## 0. はじめに

### 0.1. Dockerが必要な理由

> Docker（ドッカー）は、コンテナ仮想化を用いてアプリケーションを開発・配置・実行するためのオープンプラットフォームである。  
> ー Wikipedia「Docker」より



「コンテナ仮想化」とは、アプリが動くために必要な「ライブラリ、設定ファイル、コード」をまるごと**「イメージ」**というカタマリに詰め込む技術です。このイメージから起動した**「コンテナ」**さえあれば、どんなPCでも同じようにアプリを動かせます。

### 0.2. 事前準備



公式サイト（[docker.com](https://www.docker.com/ja-jp/)）から **Docker Desktop** をインストールしてください。

> [!IMPORTANT]
> インストール時に問題が起きた場合は、気軽に質問してください！

## 1. なぜDockerが必要なのか

### 1.0. Docker なし環境の実演
通常、新しいプロジェクトを始めるにはOSごとに異なる複雑な手順（Node.jsのインストール、DBの設定、パスの確認など）が必要で、10〜15分、あるいは数時間かかることもあります。

**「自分のPCでは動くのに、友達のPCでは動かない」**という問題は、Dockerを使えば解決します。

### 1.1. Docker の基礎知識

#### 1.1.1. Dockerfile とは
コンテナの「レシピ」です。どのOSを使い、どのファイルをコピーし、どのコマンドを実行するかを記述します。

#### 1.1.2. docker-compose とは
複数のコンテナ（フロント・バック・DB）をまとめて管理する「指示書」です。

## 2. Docker実践講習

### 2.0. サンプルアプリケーション
今回は「フロントエンド(HTML/JS)」「バックエンド(Express)」「データベース(PostgreSQL)」が連携するタスク管理アプリを動かします。

### 2.1. Docker でアプリを動かす
ディレクトリ `WebApp/` に移動して、以下の2つのコマンドを実行するだけです。

```bash
# 1. 全ての環境をビルド（レシピ通りに環境を組み立てる）
docker compose build

# 2. コンテナを起動（アプリを動かす）
docker compose up
```
